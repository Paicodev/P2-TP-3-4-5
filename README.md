### Arquitectura y Organización del Código

Para una mejor organización en el código, estructuré el proyecto bajo las sig reglas:

* **Ramas (Branches):** Aislé cada Trabajo Práctico en su propia rama (`tp3`, `tp4`, `tp5`). Para evaluar un TP específico, es necesario cambiar a su rama correspondiente.
* **Paquetes (Packages):** Dentro de cada rama, los temas están divididos en paquetes. Cada paquete representa una **Sección** puntual de la guía práctica (ej. `seccion1`, `seccion2`).
* **Archivos (Classes):** Dentro de cada paquete, divií el código para que cada **Ejercicio** tenga su propio archivo `.java` independiente (ej. `ejercicio1_1.java`, `ejercicio1_2.java`).

De esta manera, cada concepto (como lambdas, streams o hilos) puede ser evaluado y ejecutado de forma totalmente independiente sin solaparse con los demás ejercicios.
